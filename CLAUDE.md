# OpenClaw

## Project Overview
OpenClaw is an agentic development pipeline that orchestrates Grok (planning/orchestration) and Claude Code (execution) with persistent memory, granular security, quality gates, and human oversight. It turns stateless AI invocations into a coherent, auditable development workflow.

## Architecture ‚Äî The Complete Pipeline
```
You (on/off switch + approval queue)
        |
Grok (orchestration, planning, Obsidian memory R/W)
        |
update_primer.py (CLAUDE.md refresh from state.json + Obsidian)
        |
gatekeeper.py (permission classification, blocked list, mode check)
        |
git checkout -b agent/task-branch
        |
claude --print "task with full context"
        |
Quality Gate Pipeline (8 layers)
        |
state.json + Obsidian log updated
        |
git push branch -> notify you for review
```

## Core Components

### 1. CLAUDE.md Dynamic Primer
- This file is read at the start of every Claude Code session
- `update_primer.py` rewrites the "Current Sprint Context" section before each invocation
- Sources: `tasks/state.json`, git log, recent file changes
- Flow: `Grok -> update_primer.py -> CLAUDE.md updated -> claude "do task" -> informed execution`

### 2. Obsidian Memory Bridge
- Vault is the shared memory layer between Grok and Claude Code
- Grok reads decisions.md + session_log.md before tasks, writes results back after
- Claude Code reads patterns.md via bash at task start
- Human-readable, editable, with graph view for cross-project relationships

### 3. Three-Layer Memory System
- **Layer 1 ‚Äî state.json**: Structured post-invocation output (last_task, status, files_changed, next_suggested)
- **Layer 2 ‚Äî Git as ground truth**: `git diff --stat` + `git log --oneline -5` after every invocation
- **Layer 3 ‚Äî Obsidian session log**: Human-readable narrative for long-term context

### 4. Gatekeeper Security (gatekeeper.py)
- **ALLOWED_WITHOUT_CONFIRMATION**: create/read files, cargo check, cargo test, add deps
- **REQUIRE_CONFIRMATION**: delete, git push, modify src/core/, drop table
- **ALWAYS_BLOCKED**: git push --force, rm -rf /, curl|bash, chmod 777
- Modes controlled via AGENT_MODE file: `safe`, `supervised`, `autonomous`

### 5. On/Off Switch
- File-based: `/agent_control/AGENT_ENABLED`
- Kill everything: delete the file. Resume: recreate it.
- AGENT_MODE file controls permission tier globally

### 6. Sandboxed Branch Per Task
- Branch naming: `agent/{project}-{task}-{YYYYMMDD}`
- Each Claude Code instance works on its own branch
- Merge only after all quality gates pass + human review
- Git history = audit log of every agent action

## Quality Gate Pipeline (8 Layers)
```
cargo check                     -> type errors [HARD GATE]
cargo clippy -D warnings        -> antipatterns [HARD GATE]
cargo audit + cargo deny        -> CVEs + license [HARD GATE]
grep unwrap/unsafe              -> manual review queue
semgrep security rules          -> OWASP patterns [HARD GATE]
api_surface diff                -> breaking changes [HARD GATE]
cargo tarpaulin --fail-under 80 -> coverage [HARD GATE]
cargo mutants                   -> test quality [SOFT GATE]
adversarial Claude review       -> independent skeptical pass [SOFT GATE]
Grok spec-vs-implementation     -> semantic accuracy [HARD GATE]
runtime smoke test              -> actual behavior [HARD GATE]
regression comparison vs main   -> no broken tests [HARD GATE]
```
- Self-heal loop: max 3 attempts on hard gate failures
- After 3 failures: write to blocked_tasks.md, notify human

## What Always Requires Human Eyes
- **Architectural drift** ‚Äî locally correct but globally wrong decisions
- **Unsafe blocks** ‚Äî every `unsafe` block requires explicit human sign-off
